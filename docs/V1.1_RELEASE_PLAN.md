## üÜï New Features in v1.1

### 1. Extended Kalman Filter (EKF) Module
**File:** `python/filters/ekf_nonlinear.py`

- **Nonlinear Measurement Models:**
  - Radar (Range-Azimuth-Elevation)
  - Radar 2D (Range-Azimuth)
  - Bearing-only (passive tracking)
  - GPS geodetic (WGS-84)
  - ADS-B position reports

- **Coordinate Transforms:**
  - Geodetic ‚Üí ECEF
  - ECEF ‚Üí ENU (local tangent plane)
  - RAE ‚Üî Cartesian

- **EKF-IMM Integration:**
  - Combined nonlinear measurement handling with IMM maneuver detection

**Use Cases:**
- Radar tracking with nonlinear measurements
- GPS/INS fusion
- Passive bearing-only tracking

---

### 2. Track-to-Track Fusion Module
**File:** `python/fusion/track_to_track_fusion.py`

- **Fusion Methods:**
  - Covariance Intersection (CI) - unknown correlations
  - Bar-Shalom-Campo - known correlations
  - Information filter fusion
  - Weighted average

- **Track Correlation:**
  - Mahalanobis distance gating
  - Multi-hypothesis correlation
  - Union-find clustering

- **Fusion Manager:**
  - System track lifecycle management
  - Sensor-to-system track mapping
  - Stale track removal

**Use Cases:**
- Multi-radar ATC environments (ARTAS)
- Distributed sensor networks
- Heterogeneous sensor fusion (radar + ADS-B + optical)

---

### 3. ROS2 Interface
**File:** `python/interfaces/ros2_interface.py`

- **ROS2 Node:**
  - Subscribes to PointCloud2 (radar detections)
  - Publishes MarkerArray (RViz visualization)
  - Configurable parameters (dt, sigma, frame_id)

- **Standalone Interface:**
  - Works without ROS2 installation
  - Compatible API for testing
  - Batch processing support

- **Compatibility:**
  - ROS2 Humble/Iron/Jazzy
  - Autoware.Universe
  - PX4/MAVROS

**Use Cases:**
- Autonomous vehicles
- Robotics research
- Drone tracking systems

---

### 4. GPU Acceleration Module
**File:** `python/accel/gpu_acceleration.py`

- **Backends:**
  - CuPy (CUDA) - primary
  - Numba CUDA - alternative
  - CPU fallback - automatic

- **Accelerated Operations:**
  - Batched matrix multiply
  - Batched matrix inverse
  - Batched Cholesky decomposition
  - Batched linear solve

- **Batched UKF:**
  - Process 1000+ tracks in parallel
  - ~10x speedup with GPU
  - ~100,000 updates/sec (CPU baseline)

**Use Cases:**
- High-density tracking (>100 simultaneous targets)
- Real-time surveillance systems
- Batch reprocessing
